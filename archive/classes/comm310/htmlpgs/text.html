Marking up text with HTML

After pinning down the fundamental HTML "filing cabinet", it's time to move on to the markup for you text. Text is the real meat of a webpage; amidst the flashy images and cool css design, the actual text, the content you deliver is more often then not what your user have really come to see. Improper markup will make it different for them to read and understand, and will also make it difficult for them to find, as search engines are beginning to rely more and more heavily on the way you mark things up.

I'll cover the most commonly used tags here, which allow you to create paragraphs, empahasize particular words, create headings, and create lists. There are many other types of text markup, including abbreviations, acronyms, addresses, quotes, and others. I'd encourage you to refer the HTML dog text book, and other sources for more information on these.

The Paragraph Tag

We'll start with the paragraph tag. Simply put, the paragraph tags are placed around anything that is a paragraph. For example, this chunk of text you're reading right now. It's been wrapped in opening (<p>) and closing (</p>) paragraph tags...because it's a paragraph. If the chunk of text you're marking up isn't a paragraph, then don't wrap it in paragraph tags. If you want to insert a picture into your page, don't wrap it in paragraph tags either. Unless of course it's part of a paragraph. Some new designers go a little crazy and wrap everything with p tags. Don't do that. Here's a correct example of using paragraph tags:



<p>This is a paragraph. It is in between the paragraph tags.
    Isn't it fantastic? </p>

Remember: By default, paragraphs have a certain amount of padding or space above/below them. This can be changed w/ css.

Emphasis and Strong

The emphasis and strong tags are used to add a little extra umph to a word or phrase. (I added both the strong and em tags around the word "umph". By default these tags make the text look a little difference, but when talking about HTML, the point is that the words or phrases literally mean more to the browser, which means they also mean more to the search engine. Here's and example of the code:




    <p>I want <em> emphasis </em> and <strong> strength </strong>
       in this paragraph.</p> <em><strong>Lots of it.</strong></em>
    </p>

Remember: When html tags are nested, they inner tags need to be closed before outer tags, as is shown above. (i.e. - the paragraph closing tag comes after the strong and emphasis tags because it opened first. It is the "outer" or "parent" element.

Headings

Heading tags are used to identify headings, such as the word "Headings" up above this paragraph. Headings come in 6 preset sizes, 1 being the most important, and 6 the least. By deafault, your browser will display that at different sizes, but these can be altered with css. Here's how you use them:


<h1>This is the most important heading</h1>
<h2>This is the second most important heading</h2>
</h6>This is the least important.</h6>

Remember: Headings by default will take up an entire line.

Lists

There are three types of lists in html:

  • ordered lists
  • unordered lists
  • definition lists

Order lists should be used when the information you're displaying needs to be in a particular order, such as a list of chronological steps to complete. By default ordered lists show up with numbers. Unordered lists, like the one above, should be used when lists have no chronological order. By default these lists are bulleted. Definition lists should be used when defining words, so to be honest you may or may not use it quite so often. Following is an example of the code for an unordered list. To use an ordered list, simply substitue "ol" for "ul". To use a definition list, substitute "dl" for "ul" and then use "dt" for the term to be defined and "dd" for the definitions of that term.


<ul>
    <li>List item 1</li>
    <li>List item 2</li>
    <li>List item 3</li>
    <li>List item 4</li>
</ul>

Remember: Lists can be nested, this can create some nifty results when using lists for hyperlinks. The navigation at the top of my pages were created using nested unordered lists and some css.
